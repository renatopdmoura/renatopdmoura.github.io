Protótipo 1:

static uint genProgramShader(std::string vs_path, std::string fs_path)

Um gerador de programa de sombreamento de fabrica. É util para definir objetos de programas que serão compartilhados por diferentes instâncias.

Exemplo:

unsigned int program = SRW::genProgramShader("vs.glsl", "fs.glsl");

Protótipo 2:

static uint genProgramShader(std::string vs_path, std::string fs_path, std::string gs_path)

Um gerador de programa de sombreamento para situaçãos em que se faz necessário um sombreador de geometria.

Exemplo:

unsigned int program = SRW::genProgramShader("vs.glsl", "fs.glsl", "gs.glsl");

Protótipo 3:

static void genProgramShaders()

Gera os programas de sombreamentos definidos internamente.

SRW::genProgramShaders();

A chamada dessa função deve preceder o uso de qualquer program do vetor SRW::programs.

Protótipo:

static void genGeometryBuffer(uint& gBuffer, uint& gDepth, uint& gPosition, uint& gNormal, uint& gAlbedo, uint width, uint height)

Constroi texturas para serem usadas no passe de renderização como buffer de dados da geometria em cena. Esse método deve ser chamado se a intenção for usar renderização diferida.

Exemplo:

unsigned int gBuffer, gDepth, gPosition, gNormal, gAlbedo, width, height; SRW::genGeometryBuffer(gBuffer, gPosition, gNormal, gAlbedo, width, height);

Protótipo 1:

static void sharedUniforms(mat4& matrix, vec3& cameraPos, Sun* sun, std::vector* pointLightArray, std::vector* spotLightArray = NULL)

Chame essa função para definir um local de memória compartilhavel para atributos de câmera e dados de iluminação da qual todos objetos possam realizar a leitura.

Exemplo:

Camera fpsCam(vec3(0.0f, 4.0f, 0.0f), vec3(0.0f, -1.0f, -1.0f), 45.0f, 0.1f, 100.0f);
Sun mySun = {vec4(0.0f, 1.0f, 0.0f), vec4(0.2f, 0.2f, 0.2f)};
std::vector myPointLightArray = {{vec4(0.0f, 2.0f, 8.0f), vec4(20.0f, 0.0f, 0.0f)}};
std::vector mySpotLightArray = {{vec4(0.0f, 1.0f, 0.0f, 0.65f), vec4(0.0f, 4.0f, 0.0f, 0.35f), vec4(20.0f, 20.0f, 20.0f)}};
SRW::sharedUniforms(fpsCam.getViewAndProjection(), fpsCam.getPosition, &mySun, &myPointLightArray, &mySpotLightArray);

Protótipo 2:

static void sharedUniforms(mat4& viewProj, vec3& cameraPos, bool update = true)

Método para atualizar a matriz de visualização e a posição da câmera no buffer uniforme.

Essa função precisa ser chamada a cada iteração do loop de renderização.

Camera fpsCam(vec3(0.0f, 4.0f, 0.0f), vec3(0.0f, -1.0f, -1.0f), 45.0f, 0.1f, 100.0f);
while(!quit){
{
// pool events
}
SRW:sharedUniforms(fpsCam.getViewAndProjection(), fpsCam.getPosition());
}

Protótipo:

void setUniform1f(std::string, float value)

void setUniform1f(std::string name, float* value)

A primeira versão inicializa um uniforme do tipo float no programa de sombreamento atualmente vinculado usando passagem de cópia de valor - útil para uniformes excluisvos constantes.

A segunda versão aceita um endereço de memória destinada ao uso de uniformes intensivamente atualizados a cada quadro de renderização.

Protótipo:

setUniformMat4f(std::string name, mat4* value)

Define o valor de um uniforme do tipo mat4 a partir de uma referência.

Chame essa função para que alterções na matriz tomada por referência em algum momento posterior do código reflitam na entrada uniforme do sombreador.
