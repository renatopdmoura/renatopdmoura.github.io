Pseudoclasses

Pseudo classes são usadas para determinar estados especiais de um elemento.

Esses estados podem ser listados como quando:

  • O usuário sobrepoe o ponteiro do mouse sobre o elemento.
  • O usuário seleciona o elemento.
  • Um elemento está em foco.

A estrutura sintática para determinar pseudoclasses é apresentada abaixo:

                        selector:pseudo-class{
                            propierty: value;
                        }

Um uso bastante comum de pseudoclasses é para quando lidamos com links. A tag <a> aceita possuem um lista de pseduoclasses úteis para determinar o comportamento de um link dada situação:

  • link: quando um link não foi visitado pelo usuário.
  • visited: quando um link foi visitado pelo usuário.
  • hover: quando o mouse está sobre o link.
  • active: quando o usuário seleciona o link.

Um pequena implicação surge aqui. um definição de hover deve ser precedida por uma definição link e visited(se definidas). active deve ser sempre definida após hover(se definida).

Classes HTML são livres para serem combinadas com pseudoclasses.

Existem diferentes conjuntos de pseduoclasses que objetivam cumprir propósito diferentes para diferentes tags.

A seguintes pseudoclasse definem um comportamento para um elemento. Esse conjutno realiza uma analise estrutural dos elementos e retorno dados relevantes para operação. Essas pseudo-classes são listadas como:

  1. first-child
  2. last-child
  3. nth-child(n)
  4. nth-child-last-child(n)
  5. first-of-type
  6. last-of-type
  7. nth-last-of-type(n)
  8. nth-of-type(n)
  9. only-child
  10. only-of-type
  11. lang
  12. empty
  13. not

Outras pseudoclasses não categorizadas:

  1. checked
  2. disable
  3. enable
  4. focus
  5. in-range
  6. invalid
  7. optional
  8. out-of-range
  9. read-only
  10. read-write
  11. required
  12. root
  13. target
  14. valid
