CSS Book- Caixas CSS

Seletores CSS são estruturados como caixas que contém algum conteúdo. Por vezes, não estamos interessados nos limites dessas caixas e apenas queremos ver o conteúdo referido, como ocorre com o seletor p. Por outro lado, as vezes desejamos criar blocos formatados para representar cartões, artigos ou paíneis. Para realizar essa tarefa com eficiência, é preciso entender o modelo de caixas CSS para seletores. Veja:

    Todo seletor CSS(uma tag HTML) possui dimensão(podendo ser dinâmica - em, vw, vh, % - ou estática - mm, cm, px):

  • width: determina a largura, o espaço real especificado para o conteúdo do seletor.
  • height: determina a altura, o espaço real especificado para o conteúdo do seletor.
  • padding: determina o preechimento interno. Na prática, esse preenchimento e adicionado ao espaço destinado ao conteúdo, contudo, o espaço de preenchimento não contera qualquer conteúdo.
  • border: determina a borda do seletor. A borda é definida após a soma dos valores para largura e altura com o preenchimento.
  • margin: a margem é o espaço externo do seletor. Margens são transparentes e interagem com os objetos próximos(cruzando margens e.g).
  • outline: são contornos adicionados aos seletores. As dimensões de contorno de um seletor não interferem na interação objeto-objeto e podem ser visto apenas como adorno visual. Contornos partem do fim da borda e do começo das margens.

A cor preta preenche o plano de fundo do espaço do conteúdo mais o preenchimento(padding). Em magenta está a borda do seletor com 4px de espessura. Essas propriedades possuem valores próprios permitindo estilizar e controlar com maior flexibilidade a formatação dos elementos HTML. Vamos cobrir sobre nas seções seguintes.

width e height

Como já mensionado, width e height definem o espaço real do conteúdo. As unidades de media podem ser conforme descrito no capítulo Unidades de Medida.

padding

A propriedade define o preenchimento em torno do conteúdo(por isso considerado preenchimento interno). padding é uma forma abreviada que leva em conta os valores para o preenchimento superior, direito, inferior e esquerdo. Caso seja definido apenas um parâmtro, o preenchimento será o mesmo para todos os lados. padding pode ser acessado em sua forma especifica como:

  • padding-top: preencher ao topo.

    • padding-top

  • padding-right: preencher à direita

    • padding-right

  • padding-bottom: preencher à baixo

    • padding-bottom

  • padding-left: preencher à ésquerda.

    • padding-left

  • padding: preencher ao topo, direita, esquerda e inferior uniformemente.

    • padding

border

A propriedade border define bordas para um seletor. border assim como padding é uma forma abreviada da propriedade e pode ser especificada pelas variações:

  • border-top: borda superior.
  • border-right: borda direita.
  • border-bottom: borda inferior.
  • border-left: borda esquerda.

Cada especificação da propriedade border- possuem combinações para definir valores individuais como:

  • border-[top|right|bottom|left]-style: podendo ser solid, dashed, dotted.
  • border-[top|right|bottom|left]-color: definido em RGB, RGBA, HEX, HSL e HSLA.
  • border-[top|right|bottom|left]-width: a espessura da borda em mm, cm, px, %...

Uma vez que deseja-se formatar uniformemente todas as bordas, use a versão abreviada seguindo o modelo border: width style color;

margin

A propriedade margin define as margens de um seletor CSS. As margens podem ser especificadas individualmente seguindo a definição:

  • margin-top: a margem superior.
  • margin-direita: a margem direita.
  • margin-inferior: a margem inferior.
  • margin-esquerda: a margem esquerda.

outline

A propriedade outline define o contorno que envolve um seletor. Um contorno pode ser formatado a partir das propriedades variantes de outline. Tais são:

  • outline-color: a cor do contorno(RGB, RGBA, HEX, HSL, HSLA)
  • outline-width: a espessura do contorno(mm, cm, px)
  • outline-style: o estilo da linha de contorno(solid, dashed, dotted...)
  • outline-offset: o deslocamento do contorno em relação as bordas do objeto.

Exibição

Algumas tagas HTML quando tratadas apenas levam em considerção o espaço denotado pelo conteúdo envolvido pela tag. Por vezes, como quando usamos listas para construir menus, desejamos que toda formatação como a largura, altura, preenchimento, bordas e margens sejam levadas em conta em nossa formatação. Para tanto, a propriedade display apresenta os seguintes valores:

  • none: quando desejamos ocultar o objeto e desconsiderar qualquer interação deste com o meio.
  • inline: é o modo padrão para muitas das tags html. O conteúdo do elemento será avaliado como um elemento que ocupa apenas a linha em que é definido.
  • block: toda caixa CSS será levada em conta nos calculos de posicionamento. Após um elemento ser determinado como um bloco, uma quebra de linha será inserida.
  • inline-block: toda caixa CSS será levada em conta nos calculos de posicionamento, entretanto, o elemento será julgado na linha em que reside.

A principal diferença entre inline e inline-block é percebida quando ampliamos nossa página. Note que quando os elementos da <ul> quebram, para inline apenas o conteúdo direto da <li> desloca-se uma linha para baixo respeitando a altura limite dos objetos. Já para inline-block não só o espaço denotado pelo conteúdo mas também o preenchimento, bordas e margens são levados em conta de modo que não haja sobreposições entre os blocos de cada <li> na lista.

Observação

Ainda podemos ser sucintos quanto aquilo que queremos avaliar como "limites de caixa". A proprieda box-sizing determina isso dados os seguintes valores:

  • border-box: do conteúdo às bordas será avaliado como "limites de caixa".
  • margin-box: do conteúdo às margens será avaliado como "limites de caixa".
